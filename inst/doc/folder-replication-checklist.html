Folder replication checklist

Two models

Model Registry Materials
Folder-backed studies/<folder>.yml stub only Study repo: code/, data/, artifacts/
Package-backed studies/<folder>.yml stub only Study R package on GitHub

This checklist covers folder-backed studies (simple Git repo, not an R package).

Study repo layout

rep-<doi-slug>/
  replication.yml
  README.md
  registry/             # generated by prepare_folder_paper()
    replication.yml     # → sync to registry/studies/<folder>.yml
    index.csv           # → merge into registry index.csv
  code/                 # one script per table/figure
  data/                 # optional
  artifacts/            # display outputs + manifest.json
  tests/
    testthat.R
    testthat/
      test-<id>.R

Required replication.yml fields

Paper metadata

  • paper.doi — full DOI URL
  • paper.title
  • repo or paper.study_repo — GitHub slug (org/repo); inferred from folder name when it starts with rep-

Each figure or table

replications:
  - id: fig_1
    type: figure
    code: code/fig_1.R
    format: format_fig_1          # when display differs from analysis
    artifact: artifacts/fig_1.png
  - id: tab_1
    type: table
    code: code/tab_1.R
    format: format_tab_1
    data: data/myfile.dta
    artifact: artifacts/tab_1.html

Workflow from the study repo

1. Build display artifacts

Run from the study repository root (or pass its path):

library(replicateEverything)

options(
  replicateEverything.registry_root = "../registry",
  replicateEverything.use_sibling_packages = TRUE
)

build_study_artifacts(
  location = ".",
  install_deps = TRUE
)

This runs every replication, saves PNG/HTML (or RDS) under artifacts/, and writes artifacts/manifest.json.

2. Add tests

Each tests/testthat/test-<id>.R should:

  • call replicateEverything::run_replication(doi, id) (analysis object)
  • call run_replication(..., format = TRUE) when format: is set in yaml
  • compare formatted output to the committed artifact (HTML normalize or PNG md5)

Run:

testthat::test_dir("tests/testthat")

3. Validate and prepare registry stub files

prepare_folder_paper(
  location = ".",
  build_artifacts = FALSE,   # already built above
  registry_root = "../registry"
)

This runs [check_folder_replication()] and writes:

  • registry/replication.yml — copy to registry/studies/<folder>.yml
  • registry/index.csv — merge into the registry repo index.csv

4. Sync to the registry

Option A — automatic (local registry checkout):

options(replicateEverything.registry_root = "../registry")
sync_folder_paper(".")

Option B — manual: copy the two files from registry/ in the study repo into the registry repository.

Option C — one step: run prepare_folder_paper() then sync_folder_paper() (same as options A + B).

5. Tests (recommended before prepare)

testthat::test_dir("tests/testthat")

Run tests before prepare_folder_paper() so the checklist reflects a merge-ready study.

Register shortcut

prepare_folder_paper(".", registry_root = "../registry")
sync_folder_paper(".", registry_root = "../registry")
